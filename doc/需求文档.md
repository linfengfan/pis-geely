### 一、 核心架构全景图

我们将方案分为三层：**数据入口（MCP）**、**分析引擎（Rules）**、**自动化调度（Settings）**。


### 二、 模块化配置清单

#### 1. 数据入口层：`.mcp-config.json`
A 股分析必须解决数据源问题。推荐使用基于 `Python + AkShare` 的自定义 MCP Server，因为它能免费获取东方财富、同花顺的数据。

```json
{
  "mcpServers": {
    "ashare-data-provider": {
      "command": "python3",
      "args": ["-m", "mcp_server_akshare"],
      "env": {
        "AKSHARE_DATA_DIR": "./data_sink"
      },
      "description": "提供 A 股实时行情、历史财报（三大表）、股东人数及行业估值百分位"
    },
    "python-quant-engine": {
      "command": "uv",
      "args": ["run", "python-mcp-server"],
      "description": "执行复杂数学计算：DCF 贴现、杜邦分析绘图、RIM 模型演算"
    }
  }
}
```

#### 2. 分析引擎层：`.clinerules` (灵魂所在)
在这里注入你记忆中的**“股票投资框架及逻辑”**。

```markdown
# 🎭 角色定义：资深 A 股价值投资专家
你必须以“手术刀式”分析每一个 A 股标的，严禁任何情绪化评估。

## ⚔️ 强制执行逻辑
1. **数据初筛**：调用 `ashare-data-provider` 获取近 5 年财报。
   - 识别“印钞机”：经营现金流持续为正且 > 净利润。
   - 一票否决“吞金兽”：资本支出/营业收入 > 20% 且债务率持续攀升。
2. **盈利质量审计**：
   - 必须计算 $$净利润现金含量 = \frac{经营活动现金流量净额}{净利润}$$。
   - 必须检查“应收账款/营收”占比变化，识别虚增收入风险。
3. **估值红线**：
   - 根据行业周期选择模型：成熟期（银行/水电）强制使用 $$RIM$$或$$DDM$$。
   - 科技/成长：使用 $$PEG$$。
   - **买入红线**：当前股价必须低于内在价值的 30% 以上。

## 🛠 工具链路
- 所有的分析过程必须先在 `data_sink/` 生成原始数据快照。
- 最终结论必须输出为结构化 Markdown 存入 `stocks/{{CODE}}_{{NAME}}.md`。
```

#### 3. 自动化调度层：`.claude/settings.local.json`

### 三、 完整操作流程 (The Workflow)

当你想要分析“贵州茅台 (600519)”时，流程如下：

1.  **启动命令**：你在终端输入 `claude --task "对 600519 启动手术刀式财报分析"`。
2.  **上下文加载**：Claude 读取 `.clinerules` 激活专家人格，并连接 `.mcp-config.json` 中的 `ashare-data-provider`。
3.  **数据采集**：
    * AI 自动调用 MCP 接口获取 600519 的利润表、负债表、现金流量表。
    * AI 调用 `Brave Search` 获取最新的行业政策（宏观为肉）。
4.  **逻辑处理**：
    * AI 在本地运行 Python 脚本计算 ROE/ROIC 稳定性。
    * 根据你设定的公式演算：$$V = \frac{FCFF_{1}}{WACC - g}$$。
5.  **结果输出**：
    * AI 生成报告，判定它是“印钞机”还是“吞金兽”。
    * AI 自动将结果存入 `./stocks/600519_贵州茅台.md`。
    * **全局 Skill 继承**：如果你在分析中教了 AI 一种特殊的白酒行业批价对冲逻辑，`Skill Creator` 会自动将其持久化到全局配置中。

### 四、 针对你“北京房贷”财务约束的特殊调教

在 `.clinerules` 中增加一段隐私约束（AI 会在内部逻辑中权衡，但不在报告中明说）：

> **资产流动性预警**：鉴于用户存在长期固定现金流支出（房贷），在推荐重仓位标的时，必须额外进行 **“股息现金流覆盖比率”** 计算。如果标的的预期年化分红无法覆盖 3 个月的现金支出压力，必须在“风险提示”中置顶说明，优先确保组合的抗风险能力（符合鳄鱼交易体系原则 1：管控仓位不被动）。
