# 测试计划

## 1. 测试策略

- **单元测试：** Vitest（轻量、快速，支持 Vite 项目）
- **集成测试：** Vitest + React Testing Library（组件行为、Hook 测试）
- **E2E测试：** Playwright（跨浏览器自动化测试）

### 测试金字塔

```
        ▲ E2E 测试 (少量，核心流程)
       ╱ ╲
      ╱   ╲
     ╱     ╲
    ╱───集成测试───╲    (覆盖 API、组件交互)
   ╱                 ╲
  ╱───────────────────╲
 ╱      单元测试        ╲   (函数、工具类、业务逻辑)
```

## 2. 测试覆盖范围

### 2.1 核心模块

| 模块 | 测试类型 | 覆盖内容 |
|------|----------|----------|
| **估值模型计算** | 单元测试 | DDM、DCF、PE、PB-ROE、EV/EBITDA 算法验证 |
| **API接口测试** | 集成测试 | 公司管理、财务数据、宏观快照、估值、组合 |
| **前端组件测试** | 集成+E2E | MetricCard、ValuationDashboard、LiquidityMonitor |
| **风险熔断机制** | 单元测试 | 中美利差>150BP、USD/CNY贬值>1%、社融下滑>30% |
| **安全性测试** | 集成测试 | API Key 认证、速率限制、SQL 注入防护 |
| **性能测试** | E2E | 首屏加载<3s、图表渲染<1s、API响应<500ms |

### 2.2 优先级

| 优先级 | 模块 | 说明 |
|--------|------|------|
| P0 | 估值模型计算、风险熔断机制 | 核心业务逻辑，错误会导致严重后果 |
| P0 | API 接口测试 | 所有前后端数据交互 |
| P1 | 前端组件测试 | UI 交互体验 |
| P2 | 性能测试、响应式布局 | 非功能性需求 |

## 6. 测试用例示例

### 6.1 估值模型测试

```typescript
// __tests__/valuation.test.ts
import { describe, it, expect } from 'vitest'
import { calculateSafetyMargin, selectValuationModel } from '@/lib/analysis/valuationModels'

describe('安全边际计算', () => {
  it('内在价值100，当前价格70，安全边际应为30%', () => {
    const result = calculateSafetyMargin(100, 70)
    expect(result).toBe(30)
  })
})

describe('模型选择', () => {
  it('银行/保险行业应选择PB-ROE', () => {
    const model = selectValuationModel('BANK_WATER')
    expect(model).toBe('PB-ROE')
  })
})
```

### 6.2 风险熔断测试

```typescript
// __tests__/riskAlert.test.ts
describe('风险熔断机制', () => {
  it('中美利差超过150BP应触发红色预警', () => {
    const result = calculateSpreadRisk(2.31, 4.52) // CN10Y, US10Y
    expect(result.triggered).toBe(true)
    expect(result.level).toBe('red')
  })

  it('USD/CNY单日贬值超过1%应触发橙色预警', () => {
    const result = calculateForexRisk(7.20, 7.28) // yesterday, today
    expect(result.triggered).toBe(true)
    expect(result.level).toBe('orange')
  })
})
```
