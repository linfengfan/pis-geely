**日期**: 2026-05-03
**状态**: 已批准
**版本**: v1.0

---

## 1. 核心目标

构建一个**估值工具**，用户输入公司财务数据 + 宏观数据 + 估值参数，系统调用 LLM 计算股票的内在价值，返回具体的估值价格和安全边际。

**技术选型**: 所有估值计算通过 LLM（Claude API）完成，不使用传统 JS 估值公式。

## 3. API 设计

### Endpoint: `POST /api/valuation`

**Request Body**:
```json
{
  "companyName": "贵州茅台",
  "code": "600519",
  "financialData": {
    "netProfit": 823.20,
    "shares": 12.56,
    "operatingCashFlow": 615.22,
    "dividendPerShare": 30.0,
    "equity": 2446,
    "roe": 32.53
  },
  "macroData": {
    "cn10y": 1.762,
    "usdcny": 6.82,
    "socialFinancing": 7.9
  },
  "valuationParams": {
    "wacc": 6.76,
    "terminalGrowthRate": 2.5,
    "dividendRatio": 50,
    "projectionYears": 5
  },
  "currentPrice": 1680
}
```

**Response (Success)**:
```json
{
  "success": true,
  "data": {
    "intrinsicValue": 1850,
    "marginOfSafety": 10.1,
    "conclusion": "观望",
    "recommendedModel": "DCF",
    "allModels": {
      "PE": { "intrinsicValue": 1720, "marginOfSafety": 2.3 },
      "DCF": { "intrinsicValue": 1850, "marginOfSafety": 10.1 },
      "DDM": { "intrinsicValue": 1680, "marginOfSafety": 0 },
      "PB-ROE": { "intrinsicValue": 1750, "marginOfSafety": 4.0 }
    },
    "reasoning": "...",
    "calculationProcess": "..."
  }
}
```

**Response (Error - No API Key)**:
```json
{
  "success": false,
  "error": "API_KEY_NOT_CONFIGURED",
  "message": "请先在设置页面配置 API Key"
}
```

```

【宏观数据】

  "reasoning": "计算逻辑说明",
  "calculationProcess": "各模型详细计算过程"
}
```

## 5. 数据流

```
用户填写表单
    ↓
前端组装请求 JSON
    ↓
POST /api/valuation
    ↓
服务端检测 API Key
    ↓
有 Key → 组装 LLM Prompt → 调用 Claude API
无 Key → 返回错误 {"error": "API_KEY_NOT_CONFIGURED"}
    ↓
LLM 计算并返回 JSON 估值结果
    ↓
服务端解析 JSON → 返回给前端
    ↓
前端展示：内在价值、安全边际、结论、多模型对比、计算过程
```

## 8. 关键约束

1. **所有估值计算必须过 LLM**，不使用 JS 估值公式
2. **API Key 必须可配置**，不硬编码
3. **每次估值需手动输入参数**，不保存默认值
4. **返回实际计算价格**，不是 0 或估算值

---

## 9. 验收标准

- [ ] 用户可以在表单输入所有财务数据
- [ ] 用户可以输入宏观数据和估值参数
- [ ] 系统调用 LLM 计算并返回实际价格
- [ ] 返回结果包含多模型对比和安全边际
- [ ] API Key 未配置时显示提示，不阻止操作
- [ ] API Key 可以在 Settings 页面配置和保存
